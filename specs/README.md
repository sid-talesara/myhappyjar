# My Happy Jar Specs

This folder breaks the PRD into feature-level product specs for the Expo React Native v1 app.

## Files
- `00-product-foundation.md` — shared product rules, platform assumptions, data principles, navigation, and global constraints
- `01-onboarding.md` — animated onboarding experience and first-run flow
- `02-home-jar.md` — primary jar/home screen spec
- `03-add-note-sheet.md` — partial-sheet note composer and fold-drop ritual
- `04-memories-and-restricted-access.md` — resurfaced memories, archive view, and limited early access rules
- `05-shelf-and-year-end.md` — yearly rollover, shelf/archive, and completed jar experience
- `06-streaks.md` — streaks screen and streak calculation rules
- `07-profile-settings-and-privacy.md` — app lock, export/delete, settings, and sync preferences
- `08-data-model-and-offline-sync.md` — entities, local-first storage, and future sync assumptions
- `09-motion-haptics-and-sound.md` — cross-feature motion system and sensory feedback rules

## How To Use
- Start with `00-product-foundation.md`.
- Then read the feature spec for the screen or system being designed or built.
- Treat anything marked `Future` as out of scope for the v1 implementation.
